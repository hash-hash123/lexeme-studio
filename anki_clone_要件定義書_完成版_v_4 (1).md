# 多言語語彙習得プラットフォーム 要件定義書 v1.0

> **プロダクト目標**  
> 本プロダクトは、単なるAnki互換SRSではなく、**ゼロ初学者を複数言語の高水準語彙運用まで引き上げる多言語学習プラットフォーム** を目標とする。  
> 初期対象言語は **英語・ドイツ語・日本語** とし、以下の主要学習方向を最優先で支援する。  
> - ドイツ語母語話者 → 日本語 / 英語  
> - 英語母語話者 → ドイツ語 / 日本語  
> - 日本語母語話者 → ドイツ語 / 英語  
> 将来的には他言語へ拡張可能な共通基盤を前提に設計する。

> **設計方針の転換**  
> 従来の「Anki互換性中心」の設計から、**語彙習得成果中心** の設計へ移行する。  
> SRSは中核機能として維持するが、学習体験全体は以下を含む統合型とする。  
> - レベル別カリキュラム  
> - 言語ペア別出題  
> - 語彙難易度制御  
> - 文脈・例文・音声・発音支援  
> - 学習到達度測定  
> - 言語追加可能なコンテンツ基盤

---

## 0. このプロダクトが何か

本プロダクトは、**外国語の語彙力をゼロからネイティブ相当の高水準まで体系的に伸ばす学習サービス** である。

これは単なる暗記カードアプリではない。  
SRS、カリキュラム、語彙レベル設計、例文、音声、言語ペア別UI、習熟判定を一体化し、学習者が以下を段階的に獲得できるようにする。

1. 初級語彙の認識
2. 基本語彙の運用
3. 中上級語彙の精密理解
4. 学術・文学・口語・新聞・抽象概念語彙の運用
5. ネイティブ相当の語彙理解と使い分け

## 1. プロダクトの中核価値

### 1.1 解決する課題

既存の単語帳やSRSアプリには以下の問題がある。

- 初学者にとって何から始めるべきか分からない
- 母語によって理解しやすい説明の形が異なる
- 単語だけ覚えても使えるようにならない
- 中級以降、抽象語彙・多義語・語感差・レジスター差が学びにくい
- 日本語・英語・ドイツ語のように構造差が大きい言語間で、同一UIでは学びにくい
- 言語追加時にコンテンツ構造が破綻しやすい

### 1.2 提供価値

本プロダクトは以下を提供する。

- 学習者の母語に最適化された語彙提示
- 目標言語ごとの段階的カリキュラム
- SRSによる長期記憶定着
- 文脈・例文・音声・品詞・語法・コロケーションを含む深い語彙学習
- CEFR/JLPT等に近いレベル設計との接続
- 言語追加しやすい共通データモデル

## 3. 到達目標

### 3.1 学習到達の定義

本プロダクトは「単語を見て意味が分かる」だけではなく、以下を段階的に到達目標とする。

1. **認識**: 単語を見聞きして意味が分かる
2. **想起**: 母語から目標言語へ思い出せる
3. **識別**: 類義語や多義語を区別できる
4. **運用**: 文脈の中で適切に使える
5. **精密理解**: レジスター、語感、コロケーション、文法制約を理解できる
6. **ネイティブ相当語彙理解**: 高頻度語から抽象語・専門語・慣用表現まで読解可能

### 3.2 レベル体系

初期は内部レベルとして以下を持つ。

| レベル | 目安 |
|--------|------|
| L0 | 完全初学者 |
| L1 | 超初級 |
| L2 | 初級 |
| L3 | 初中級 |
| L4 | 中級 |
| L5 | 中上級 |
| L6 | 上級 |
| L7 | ネイティブ近接 |
| L8 | ネイティブ相当語彙読解 |

外部基準とのマッピングは参考情報として保持する。

- 英語 / ドイツ語: CEFR A1〜C2相当
- 日本語: JLPT N5〜N1相当 + N1超語彙層

## 4. 学習体験の柱

### 4.1 SRS

- 忘却曲線に基づく復習スケジューリング
- 認識カード・想起カード・聴解カード・例文穴埋めカードを統合管理
- 学習ログに基づく難易度再調整

### 4.2 カリキュラム

- 完全初学者向けの順序付きコース
- レベル別語彙セット
- テーマ別語彙セット（日常、旅行、ビジネス、学術など）
- 母語別に説明内容を最適化

### 4.3 深い語彙学習

各語彙項目に対して以下を扱う。

- 見出し語
- 品詞
- 意味
- 母語訳
- 補助説明
- 例文
- 音声
- 発音表記
- コロケーション
- 類義語 / 反意語
- 使用レジスター
- 多義語区分
- 学習上の注意点

### 4.4 言語ペア最適化

同じ語彙でも、母語によって説明を変える。

例:
- 日本語話者向け英語: 冠詞、可算不可算、語法差を強調
- 英語話者向け日本語: 敬語、助詞、漢字、訓読を強調
- ドイツ語話者向け英語: 語順や前置詞、偽友達を強調

## 6. コンテンツ設計

### 6.1 中核エンティティ

従来の Anki 的 `note/card/template` 構造だけでは不足するため、以下の概念を持つ。

- **Language**: 言語
- **LanguagePair**: 母語 × 学習言語
- **Lexeme**: 語彙項目の中核概念
- **Sense**: 語義
- **Form**: 活用形・表記揺れ・見出し形
- **ExampleSentence**: 例文
- **Course**: カリキュラム
- **Lesson**: コース内レッスン
- **StudyItem**: 学習単位
- **CardVariant**: 出題形式
- **ReviewLog**: 復習ログ

### 6.2 Lexeme中心設計

1つの語彙項目に複数の意味・表記・説明を紐づける。

例:
- 日本語: 行く / いく / 行きます
- 英語: run（走る / 運営する / 流れる）
- ドイツ語: gehen / ging / gegangen

これにより、単純な表裏カードではなく「語彙概念」を中心に複数カードを生成できる。

### 6.3 CardVariant

同じ語彙から以下のようなカードを生成する。

- 認識カード: 目標言語 → 母語
- 想起カード: 母語 → 目標言語
- 音声認識カード: 音声 → 意味
- 例文穴埋めカード
- 多義語識別カード
- 類義語比較カード
- 漢字 / 綴り確認カード

## 7. 母語別最適化

### 7.1 言語ペア前提の設計

本プロダクトは「学習言語だけ」を見るのではなく、**母語 × 学習言語** を第一級概念として持つ。

必要理由:
- 翻訳が変わる
- 説明方法が変わる
- 躓きやすい点が変わる
- 出題形式の最適解が変わる

### 7.2 例

#### 日本語話者 → 英語
- 冠詞
- 単複
- 時制のニュアンス
- 前置詞
- 語法

#### 英語話者 → 日本語
- 漢字
- かな
- 助詞
- 敬体 / 常体
- 自他動詞

#### ドイツ語話者 → 日本語
- 助詞
- 敬語
- 漢字
- 語順差

#### 英語話者 → ドイツ語
- 格変化
- 名詞性
- 分離動詞
- 語順

## 10. 技術方針

### 10.1 基本方針

- Webアプリとして提供
- SRSエンジンは独立モジュール化
- コンテンツ管理と学習ログを分離
- 言語追加可能な正規化データモデル
- UI文言はi18n対応

### 10.2 コンテンツ拡張性

新規言語追加時に必要なのは以下で済む構造を目指す。

- Language追加
- Lexeme / Sense / Form データ投入
- LanguagePair別訳文 / 説明追加
- 音声追加
- コース追加
- 必要なら専用出題形式追加

### 11.2 旧Anki型モデルとの関係

Anki型の `notes/cards` は内部実装参考にはなるが、本プロダクトでは **語彙概念中心モデル** に置き換える。  
理由は以下である。

- 多義語を扱いやすい
- 母語別訳を分離しやすい
- 複数出題形式を統合しやすい
- 将来の多言語拡張に強い

## 12. 学習アルゴリズム方針

### 12.1 段階

初期はSM-2系でもよいが、将来的には学習ログに基づく最適化を見据える。

### 12.2 学習単位別状態

語彙全体ではなく **StudyItem / CardVariant 単位** で状態を持つ。

例:
- 意味認識は得意
- スペリング想起は弱い
- 音声認識は未定着

この差を別状態として管理する。

### 12.3 出題最適化

- 直近誤答を優先
- 学習目標レベルに合わせた語彙比率調整
- 新出 / 復習 / 苦手再挑戦の配分調整
- 認識偏重ユーザーには想起カード比率を増やす

## 16. 設計上の重要結論

このプロダクトを成功させるには、設計の中心を **Anki互換カード** ではなく、**語彙概念・言語ペア・習熟状態** に置く必要がある。

したがって、今後の設計原則は以下とする。

1. 中心エンティティは card ではなく lexeme / sense / study_item とする
2. 母語 × 学習言語を第一級概念として持つ
3. SRSは機能の一部であり、カリキュラムと到達設計を同格に扱う
4. 言語追加可能性を常に優先する
5. 初期は語彙に集中し、文法・会話は後続拡張とする
